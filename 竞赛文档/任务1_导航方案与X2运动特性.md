# 任务1 定向移动 —— 导航方案与 X2 运动特性

> 代码: `usercode/py/walk_forward_steps.py`
> 诊断脚本: `usercode/py/diag_velocity.py`、`usercode/py/diag_turn.py`
> 目标: 从【出发区】自主导航到【交互区-I】，双脚完美入圈、正面朝向【交互区-II】、不撞围挡、用时 < 3 分钟。
> 实测终点: x≈0、y≈1.69、yaw≈-93°，双脚对称居中入圈（视觉确认完全入圈）。

---

## 一、场地坐标（odom 世界系）

来源 `sim_mujoco/configuration/robot/lx2501_3_t2d5/model_info/scene/room/room.xml`。
odom 话题 `/aima/hal/odom/state` 与 MuJoCo 世界系对齐（起点实测 (-1.5,-1.5) 与场地标记一致）。

| 元素 | 坐标 | 说明 |
|---|---|---|
| 场地围挡 | x∈[-2,2], y∈[-2,2] | 墙半厚 0.15 |
| **顶墙内表面 (+y侧)** | **y ≈ 1.85** | 墙心 y=2；撞到判 0 分，最危险的边 |
| 出发区 | 中心 (-1.5,-1.5) r=0.25 | 机器人起点 |
| **交互区-I** | **圆心 (0, 1.7) r=0.25** | 任务1 目标区，需双脚入圈 |
| 交互区-II | 圆心 (0, 1.0) r=0.25 | 到位后需正面朝向它 → 最终 yaw≈-90° |
| 作业区 / 作业桌 | (1,-1.5) / (1.5,-1.4) | 本任务不涉及 |

> **核心张力**：交互区-I 圆心 y=1.7 距顶墙内面 1.85 只有 0.15m。正向高速走进去，机身/脚会探到墙 → 撞墙 0 分。

## 二、X2 仿真运动控制实测特性（坑）

控制方式：发 `McLocomotionVelocity`(forward/lateral/angular) 到 `/aima/mc/locomotion/velocity`；
启动序列：`STAND_DEFAULT` → 注册输入源(priority 40) → `LOCOMOTION_DEFAULT` → 等 odom。
以下用 `diag_velocity.py` / `diag_turn.py` 量出，是全部调参的根本依据：

| 特性（实测） | 数据 | 对策 |
|---|---|---|
| **`lateral_velocity` 完全失效** | 命令横移机器人不动/只前蹭 | 不能螃蟹步；横向纠偏只能靠**转向（航向跟线）** |
| **前进天然右偏 + 偏航** | 前进伴 ~20% 右漂、~-4.5°/s 顺时针 | 直行实时锁航向；跟线用 **P+积分**消除恒定漂移的稳态误差 |
| **原地转身平移漂移大且随机** | 0.1~0.5m，方向不固定；带惯性转更猛 | **转身前 settle 停稳**(~0.8s)压到~0.1m；大转身**绝不放终点/近墙** |
| **冷启动第一步猛甩** | ~0.4m | 启动后先**热身小步** |
| **倒走慢 + 步态死区** | 上限~0.12m/s；小速度(<0.1)原地蹭卡死 | 倒走**速度下限 0.13**；倒走+跟线时**关积分**(防绕死拐偏鼻子) |
| 前进有效速度 | 命令0.2 → 实测~0.23m/s | — |

## 三、最终导航方案（五阶段）

核心思想：**把不稳定的 180° 大转身放在安全低位完成；最后一段朝着最终朝向 -90° 倒走进圈，走完即终态，不再有任何转身去破坏定位。**

```
热身小步（避开冷启动甩动）
Stage 1: 朝 0° 原地把 x 摆正
Stage 2: 转 +90° 正向快走到中途点 y = target_y - 0.8   ← 航向跟线(P+积分)压 x 漂移
Stage 3: 在安全低位 y≈0.9 转 180° 到最终朝向 -90°        ← 大转身漂移在此无害、离顶墙远
Stage 4: 朝 -90° 倒走最后 0.8m 进圈                       ← 航向跟线(P,关积分)对正 x；速度下限防卡死
Stage 5: 小角度回正锁朝向 -90°（正面朝交互区-II）
（每次转身前 settle ~0.8s 停稳）
```

**航向跟线（cross-track steering）**：机器人不能横移，故用"转向"把自己拉回目标轴线 x=target_x。
正向段用 P+积分（积分消除前进天然漂移的稳态误差）；倒走段只用 P（关积分防绕死卡死）。

## 四、关键参数（已设为默认值）

| 参数 | 默认值 | 理由 |
|---|---|---|
| `--target-x` | **-0.08** | 补偿末态系统性 +x 残差(~0.07)，使落点≈0。X2 站姿宽~0.3m，几乎填满直径 0.5m 的圈，必须居中两脚才都入圈 |
| `--target-y` | **1.70** | 正对交互区-I 圆心，整只脚(含脚角)入圈；背对墙站立后背约 1.78，离顶墙 1.85 还有 ~7cm |
| `--forward-speed` | 0.35 | 兼顾速度与漂移 |
| `--turn-speed` | 0.4 | — |
| `backward_approach` | 0.8 (代码内) | 末段倒走距离；给 P 跟线足够距离收敛 x、鼻子回正，且大转身落在更低更安全的 y |
| `cross_track_kp / ki` | 1.6 / 0.8 | 航向跟线增益 |
| `min_back_speed` | 0.13 | 倒走速度下限，避开步态死区 |
| `settle_time` | 0.8 | 转身前停稳秒数 |

## 五、运行步骤

1. 启动平台：`./start_AllinOne.sh start`（仿真 sim_mujoco + 运动控制 mc 各开一个窗口，并自动切 STAND）。
   - 其它子命令：`stop` 停容器、`restart` 停容器后重开并重启、`clean` 删容器；**无参数会报错**（必须显式指定）。
2. **在仿真窗口手动点 Reset**：机器人初始是断电跪坐，Reset 后才会在出发区站好（自动切的 STAND 会让它立即站起）。
3. 运行导航（最优参数已是默认值，一键即可）：
   ```bash
   ./run.sh usercode/py/walk_forward_steps.py
   ```
   - 需要覆盖参数时再显式传，如 `--forward-speed 0.5`、`--target-y 1.6`、`--reset-prompt`（详见脚本 `--help` 或文件头 brief）。

> ⚠️ **重复性**：停稳后机器人有 ~0.08m 站定微调，每跑落点 ±几 cm 随机浮动，不保证每次完美居中（-0.08 偏置让其平均落圆心）。正式比赛建议多跑确认；取两次最好成绩。
> ⚠️ 比赛禁止复制粘贴代码到终端、禁用云算力、需双机位监考、全程匿名。
